src/data/questions.ts 파일을 읽고 모든 문제의 정답 정확성을 검증해줘.

**검증 범위:**
`$ARGUMENTS`에 카테고리가 지정된 경우 (`한국사`, `과학`, `지리`, `일반상식` 중 하나) → 해당 카테고리만 검증
`$ARGUMENTS`가 비어 있으면 전체 40문제를 모두 검증

검증 시작 전에 "검증 카테고리: {카테고리명 또는 '전체'} / 대상 문제 수: {n}개"를 출력해줘.

## 1. 구조 무결성 검사

각 문제에 대해 아래 항목을 확인해줘.

- `id` 중복 여부 (중복 시 ❌ 표시)
- `correct_index`가 0~3 범위인지 (범위 초과 시 ❌)
- `options` 배열 길이가 정확히 4인지 (아니면 ❌)
- `question_text`, `explanation`, `category`, `difficulty` 필드가 비어 있지 않은지 (비어 있으면 ❌)
- 문제가 없으면 "✅ 구조 무결성 이상 없음" 출력

## 2. 정답-해설 일치 검사

각 문제의 `explanation`이 `correct_index`에 해당하는 보기를 실제로 지지하는지 확인해줘.

판정 기준:
- explanation에 정답 보기의 핵심 키워드가 포함되어 있거나, 정답 보기를 명시적으로 언급하면 → ✅ 일치
- explanation이 다른 보기를 설명하거나 정답 보기를 전혀 언급하지 않으면 → ⚠️ 불일치 의심

불일치 의심 문제는 아래 형식으로 출력해줘.
- ID / 카테고리
- 문제 텍스트
- 정답 보기: `correct_index`번 — "{보기 텍스트}"
- 현재 해설: "{explanation}"
- 판단 근거 및 수정 제안

## 3. 보기 품질 검사

각 문제의 보기 4개를 확인해줘.

- 보기 중 중복된 텍스트가 있으면 ⚠️ 표시
- 보기가 지나치게 짧거나(2글자 이하) 형식이 다른 보기와 크게 다르면 ⚠️ 표시
- 문제가 없으면 "✅ 보기 품질 이상 없음" 출력

## 4. 종합 요약

| 항목 | 정상 | 경고/오류 |
|------|------|-----------|
| 구조 무결성 | n개 | n개 |
| 정답-해설 일치 | n개 | n개 의심 |
| 보기 품질 | n개 | n개 |

마지막으로 즉시 수정이 필요한 문제(❌)와 검토 권장 문제(⚠️)를 ID 목록으로 정리해줘.
