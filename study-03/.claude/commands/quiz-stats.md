src/data/questions.ts와 src/store/quizStore.ts 파일을 읽고 퀴즈 게임의 전체 통계를 분석해줘.

**분석 범위:**
`$ARGUMENTS`에 카테고리가 지정된 경우 → 해당 카테고리에 대한 심층 분석 추가
`$ARGUMENTS`가 비어 있으면 전체 통계 리포트 출력

## 1. 문제 구성 현황

전체 문제 수와 카테고리별 문제 수를 표로 정리해줘.

| 카테고리 | 문제 수 | 비율 | 난이도 분포 (easy / medium / hard) |
|---------|--------|------|-----------------------------------|
| 한국사  |        |      |                                   |
| 과학    |        |      |                                   |
| 지리    |        |      |                                   |
| 일반상식|        |      |                                   |
| **합계**|        |      |                                   |

적정 난이도 비율(easy 30% / medium 40% / hard 30%)과 비교해서 카테고리별로 치우침 여부를 알려줘.

## 2. 정답 인덱스 분포

전체 문제의 `correct_index` 분포를 집계해줘.

- 0번 보기가 정답: n개 (n%)
- 1번 보기가 정답: n개 (n%)
- 2번 보기가 정답: n개 (n%)
- 3번 보기가 정답: n개 (n%)

특정 번호에 정답이 40% 이상 몰려 있으면 ⚠️ 경고로 표시해줘.

## 3. 점수 체계 분석

quizStore.ts의 `calcPoints` 함수와 `finishQuiz` 보너스를 읽고 점수 구조를 분석해줘.

| 조건 | 획득 점수 |
|------|-----------|
| 정답 | +10점 |
| 스피드 보너스 (10초 이내) | +5점 |
| 3연속 콤보 | +10점 |
| 카테고리 만점 보너스 | +20점 |

그리고 아래 시나리오별 이론적 최고 점수를 계산해줘.
- 카테고리 모드 (10문제): 전 문제 스피드+콤보 달성 시 최고 점수
- 전체 도전 (40문제): 전 문제 스피드+콤보 달성 시 최고 점수

## 4. 밸런스 진단 및 추천

아래 항목을 기준으로 현재 문제 세트의 밸런스를 진단하고 개선 제안을 해줘.

1. **난이도 밸런스**: 카테고리별 난이도 편차가 20%p 이상이면 조정 대상 카테고리 명시
2. **정답 편향**: 특정 인덱스 집중 여부 및 수정 대상 문제 ID 목록
3. **문제 수 균형**: 카테고리 간 문제 수 차이가 있으면 보완이 필요한 카테고리 제안
4. **점수 구조**: 현재 보너스 배율이 적절한지 의견 제시 (선택 시 유리한 전략이 너무 단순하지 않은지)
