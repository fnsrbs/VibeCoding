`teacher-data/rankings.json`을 읽고 학생 성적 추이와 성장을 분석해줘.

파일이 없으면 즉시 중단하고 "/teacher-import를 먼저 실행하세요." 안내를 출력해.

---

## 분석 대상 결정

`$ARGUMENTS`에 닉네임이 있으면 → 해당 학생 개인 심층 분석
`$ARGUMENTS`가 비어 있으면 → 전체 학생 성장 현황 요약

`savedAt` 타임스탬프를 기준으로 시간순 정렬해서 사용해줘.

---

## 케이스 A: 개인 심층 분석 (닉네임 지정 시)

닉네임이 데이터에 없으면 "해당 학생 기록이 없습니다." 출력 후 종료.

### A-1. 전체 플레이 이력

```
[{닉네임}] 전체 플레이 기록 (시간순)
  #1  {날짜}  {카테고리}  {점수}점
  #2  {날짜}  {카테고리}  {점수}점
  ...
  총 {n}회 플레이 / 최초 참여: {날짜} / 최근 참여: {날짜}
```

### A-2. 카테고리별 성장 곡선

동일 카테고리의 플레이 횟수가 2회 이상인 카테고리를 대상으로 출력해줘.

```
[한국사] 성장 추이
  1회: {점수}점 ({날짜})
  2회: {점수}점 ({날짜}) → {+n 또는 -n}점
  3회: {점수}점 ({날짜}) → {+n 또는 -n}점
  최고점: {점수}점 / 최근점: {점수}점 / 전체 변화: {+n 또는 -n}점
```

플레이 횟수가 1회뿐인 카테고리는 "데이터 부족 — 재도전 권장" 표시.

### A-3. 성장 종합 평가

```
전체 성장 방향: {상승 / 유지 / 하락} (전체 점수 평균 기준)
가장 많이 성장한 카테고리: {카테고리명} (+{n}점)
가장 많이 하락한 카테고리: {카테고리명} (-{n}점) 또는 "없음"
아직 도전 안 한 카테고리: {목록} 또는 "없음"
연속 플레이 기록: 최대 {n}일 연속 (기간: {시작일} ~ {종료일})
```

### A-4. 다음 목표 제안

```
[다음 플레이 제안]
  우선 카테고리: {카테고리명}
  이유: {최근 하락 / 미도전 / 가장 가능성 있는 성장 카테고리}
  목표 점수: {현재 최고점 + 10점 또는 카테고리 평균 이상}
```

## 케이스 B: 전체 학생 성장 현황 (닉네임 미지정 시)

### B-1. 최근 7일 활동 현황

오늘 날짜 기준으로 최근 7일 이내 `savedAt` 기록을 집계해줘.

```
[최근 7일 활동]
  총 플레이 횟수: {n}회
  활동 학생 수: {n}명
  비활동 학생 수: {n}명
  가장 많이 플레이한 학생: {닉네임} ({n}회)
```

### B-2. 전체 학생 성장 요약표

학생마다 첫 번째 기록과 가장 최근 기록의 점수(동일 카테고리 기준 가장 최근 vs 최초)를 비교해줘.
비교 가능한 카테고리가 없는 학생은 "신규 참여" 표시.

| 닉네임 | 비교 카테고리 | 최초 점수 | 최근 점수 | 변화 | 추세 |
|--------|------------|---------|---------|------|------|

추세: +{n}점 이상이면 📈 성장, 0±5점이면 ➡️ 유지, -{n}점 이하이면 📉 하락

### B-3. 반 전체 성장 통계

```
성장 학생 수 (📈): {n}명
유지 학생 수 (➡️): {n}명
하락 학생 수 (📉): {n}명
비교 불가 (신규): {n}명

가장 큰 성장: {닉네임} (+{n}점, {카테고리})
가장 큰 하락: {닉네임} (-{n}점, {카테고리}) — 개인 면담 권장
```
