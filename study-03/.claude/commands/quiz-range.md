src/data/questions.ts 파일을 읽고, 지정된 ID 범위의 문제를 검토해줘.

**범위 파싱:**
`$ARGUMENTS`에서 공백으로 구분된 두 숫자를 추출해 시작 ID와 끝 ID로 사용해줘.
예) `$ARGUMENTS`가 `"11 20"`이면 ID 11번부터 20번까지 검토.

## 1. 문제 목록

범위 내 각 문제를 아래 형식으로 나열해줘.

| ID | 카테고리 | 난이도 | 문제 | 정답 번호 (0-based) | 정답 보기 |
|----|---------|--------|------|---------------------|-----------|

## 2. 난이도 분포

범위 내 문제의 난이도를 집계해줘.

- easy: n개 (n%)
- medium: n개 (n%)
- hard: n개 (n%)

적정 분포(easy 30% / medium 40% / hard 30%)와 비교해서 치우침이 있으면 알려줘.

## 3. 정답 분포

`correct_index` 값(0~3)별 개수를 집계해줘.

- 0번 (1번 보기): n개
- 1번 (2번 보기): n개
- 2번 (3번 보기): n개
- 3번 (4번 보기): n개

특정 번호에 정답이 몰려 있으면 (전체의 40% 이상) 경고로 표시해줘.

## 4. 종합 의견

난이도·정답 분포 검토 결과를 한 문단으로 요약하고, 조정이 필요한 부분을 구체적으로 제안해줘.
