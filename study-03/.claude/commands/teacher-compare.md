`teacher-data/rankings.json`을 읽고 학생들을 상세 비교해줘.

파일이 없으면 즉시 중단하고 "/teacher-import를 먼저 실행하세요." 안내를 출력해.

---

## 비교 대상 결정

`$ARGUMENTS`에 닉네임이 공백으로 구분되어 있으면 → 해당 학생들만 비교
`$ARGUMENTS`가 비어 있으면 → 종합 점수 상위 3명 자동 선택

선택된 학생이 데이터에 존재하지 않으면 해당 이름을 목록으로 알려주고, 존재하는 학생만으로 진행해줘.
존재하는 학생이 1명 미만이면 즉시 중단.

---

## 1. 점수 비교표

카테고리별 최고점 기준으로 비교해줘. (미참여 카테고리는 `-`)

| 항목          | {학생A} | {학생B} | {학생C} |
|--------------|--------|--------|--------|
| 종합 점수     |        |        |        |
| 한국사        |        |        |        |
| 과학          |        |        |        |
| 지리          |        |        |        |
| 일반상식      |        |        |        |
| 전체도전(all) |        |        |        |
| 참여 카테고리 수|       |        |        |
| 총 플레이 횟수 |        |        |        |

각 행에서 가장 높은 점수 칸에 **굵게** 표시해줘.

---

## 2. 카테고리별 격차 분석

각 카테고리에서 최고점과 최저점의 차이를 계산해줘.

| 카테고리 | 최고점 (학생명) | 최저점 (학생명) | 격차 | 평가 |
|---------|--------------|--------------|------|------|

평가 기준:
- 격차 0~20점: "비슷한 수준"
- 격차 21~50점: "약간 차이"
- 격차 51점 이상: "큰 차이 ⚠️"

---

## 3. 학생별 강점 / 약점 카테고리

각 학생에 대해 출력해줘.

```
{닉네임}
  강점: {가장 높은 점수 카테고리} ({점수}점) — 반 평균 대비 +{n}점
  약점: {가장 낮은 점수 카테고리} ({점수}점) — 반 평균 대비 -{n}점
  미참여: {아직 하지 않은 카테고리 목록, 없으면 "없음"}
  성장 추이: {동일 카테고리에서 가장 오래된 기록 → 최신 기록 점수 변화, 기록이 1개면 "데이터 부족"}
```

여기서 "반 평균"은 `rankings.json`에 있는 전체 학생 기준으로 계산해줘.

---

## 4. 종합 비교 의견

비교 결과를 바탕으로 선생님에게 전달할 한 줄 요약을 각 학생마다 작성해줘.

예시 형식:
```
{학생A}: 한국사·지리에서 두각을 보이나 과학 보완 필요. 전체도전 도전 권장.
{학생B}: 전 카테고리 고른 참여. 점수 향상을 위해 스피드 보너스 전략 안내 권장.
```
