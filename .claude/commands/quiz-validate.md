src/data/questions.ts 파일을 읽고, 문제 텍스트(question_text)에 모호한 표현이 포함된 문제를 찾아 검증해줘.

**검증 범위:**
- `$ARGUMENTS`에 카테고리가 지정된 경우 (`한국사`, `과학`, `지리`, `일반상식` 중 하나) → 해당 카테고리 문제만 검증
- `$ARGUMENTS`가 비어 있으면 전체 문제를 모두 검증

**모호한 표현 목록:** `가장`, `최초`, `최대`, `최소`, `최고`, `최장`, `최단`, `유일`, `제일`

**출력 형식:**
검증 시작 전에 "검증 카테고리: {카테고리명 또는 '전체'} / 대상 문제 수: {n}개"를 먼저 출력해줘.

각 문제마다 다음 항목을 표시해줘.
- id / 카테고리 / 난이도
- 문제 텍스트 (모호한 표현 강조)
- 발견된 표현과 현재 explanation에 측정 기준·시점·범위가 명시되어 있는지 여부 (✅ 명시됨 / ⚠️ 기준 없음)
- 기준이 없으면 어떤 기준(예: 면적 기준, 2024년 기준, 세계 기준 등)을 question_text에 추가하면 좋을지 구체적으로 제안

마지막에 전체 요약(대상 문제 수, 모호한 표현 있는 문제 수, 그 중 기준 명시된 수 / 기준 없는 수)을 표시해줘.
